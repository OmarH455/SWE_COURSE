# Requirements Prompt

**Project Stage:** Requirements Specification & System Architecture  
**Goal:** Define the core functionality, user roles, and technology stack for the "Khalsny" Service Booking App.

---

## Refined Prompt (Final Version)

**Role:** You are a Senior Software Architect.

**Task:** Create a comprehensive Requirements Specification for "Khalsny", a mobile marketplace connecting clients with local service providers (cleaners, plumbers, etc.).

**Core Tech Stack:** Flutter (Frontend) + Firebase (Backend & Auth).

**Users:**
1. **Client:** Browses categories, books services, chats with providers, and tracks history.
2. **Provider:** Manages profile, accepts bookings, and sets service pricing.

**Key Features to Define:**
- Email/Password Authentication.
- Service catalog with search and categories.
- Complete Booking flow (Selection -> Confirmation -> History).
- In-app Chat system between parties.

**Output:** A prioritized feature list and high-level architecture overview.

---

## Prompt Techniques Used

- **Role Prompting:** Validates the persona of a "Senior Software Architect" to ensure high-level structural thinking.
- **Context Setting:** Clearly defines the tech stack (Flutter/Firebase) to narrow the scope of the requirements.
- **Task Decomposition:** Breaks down the request into specific user roles (Client/Provider) and functional categories.
- **Output Specification:** Explicitly requests a prioritized feature list and architecture description.
